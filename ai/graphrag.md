# 🕸️ GraphRAG 介绍

## 🤔 什么是 GraphRAG？

GraphRAG（Graph Retrieval-Augmented Generation）是微软研究院于 2024 年提出的一种增强型 RAG 技术框架 🚀，它将**知识图谱（Knowledge Graph）** 与传统的检索增强生成（RAG）相结合，大幅提升了大语言模型在复杂文档理解和全局推理任务上的能力 💪。

---

## 😤 传统 RAG 的局限

传统 RAG 流程：

```
文档 → 分块（Chunk） → 向量化 → 向量数据库 → 相似度检索 → LLM 生成
```

**核心问题：**

| 问题 | 说明 |
|------|------|
| 🔍 局部视野 | 每次只检索语义相近的几个片段，无法感知全局结构 |
| 🔗 上下文断裂 | 跨文档、跨段落的关联信息容易丢失 |
| 🌐 全局查询失效 | 对"整体主题是什么"、"所有实体的关系如何"等问题回答质量差 |

---

## 💡 GraphRAG 的核心思路

GraphRAG 在索引阶段引入了图结构，将文档中的**实体（Entity）** 和**关系（Relationship）** 提取出来，构建知识图谱 🗺️，并在此基础上进行社区划分和摘要生成。

### 🏗️ 索引流程

```
📄 原始文档
   ↓
✂️  文本分块（Text Chunks）
   ↓
🤖 实体 & 关系抽取（LLM 驱动）
   ↓
🕸️ 知识图谱构建（Entity Graph）
   ↓
🔬 图社区检测（Leiden 算法）
   ↓
📝 社区摘要生成（Community Summaries）
   ↓
📚 多层级索引结构
```

### 🔎 查询流程

GraphRAG 提供两种查询模式：

**1. 🌍 Global Search（全局搜索）**
- 适合：宏观问题，如"文档集的核心主题是什么？"
- 方式：遍历所有社区摘要 → Map-Reduce 聚合 → LLM 综合回答

**2. 📍 Local Search（局部搜索）**
- 适合：具体实体问题，如"张三和李四有什么关联？"
- 方式：定位相关实体 → 图邻居扩展 → 结合原文片段回答

---

## 🧩 关键技术组件

### 1. 🏷️ 实体与关系抽取
使用 LLM 从文本中识别命名实体（人物、组织、地点、概念等）及其关系，形成三元组：

```
(实体A) --[关系]--> (实体B)
例：(OpenAI) --[开发了]--> (GPT-4)
```

### 2. 🔬 图社区检测
使用 **Leiden 算法** 对知识图谱进行层次化社区划分，将语义相关的节点聚合为"社区（Community）" 👥。

```
🔵 Level 0：细粒度社区（小团体）
🟡 Level 1：中粒度社区
🔴 Level 2：粗粒度社区（大主题）
```

### 3. 📋 社区摘要
对每个社区调用 LLM 生成结构化摘要，作为全局查询的检索单元，解决传统 RAG 无法感知全局的问题 ✅。

### 4. 📌 协变量（Covariates）
可选地提取文本中的**声明（Claims）** 信息，如事实陈述、时间属性等，丰富图的语义信息 🌱。

---

## ⚔️ GraphRAG vs 传统 RAG 对比

| 维度 | 传统 RAG | GraphRAG |
|------|----------|----------|
| 📦 索引结构 | 向量数据库（扁平） | 知识图谱 + 向量（层次化） |
| 🌐 全局理解 | ❌ 弱 | ✅ 强（社区摘要） |
| 🔗 实体关系推理 | ❌ 弱 | ✅ 强（图遍历） |
| 💰 索引成本 | 低 | 高（需多次 LLM 调用） |
| ⚡ 查询延迟 | 低 | 全局查询较高 |
| 🎯 适用场景 | 事实检索、问答 | 分析性问题、主题归纳 |

---

## 🌟 典型应用场景

- 🏢 **企业知识库分析**：理解大型内部文档库的整体结构和主题
- 🎓 **学术文献综述**：自动梳理领域内研究脉络和学者关系
- ⚖️ **法律文档分析**：识别案件中涉及的实体关系链条
- 📰 **新闻事件追踪**：跨文章理解事件发展和人物关联
- 💻 **代码库理解**：构建模块、函数、依赖关系图谱

## 🚀 快速上手

微软已开源 GraphRAG 实现，几步即可跑起来 🎉：

```bash
# 📦 安装
pip install graphrag

# 🗂️ 初始化工作区
python -m graphrag.index --init --root ./my_project

# ⚙️ 配置 settings.yml（设置 LLM API Key 等）

# 🏗️ 构建索引
python -m graphrag.index --root ./my_project

# 🌍 全局查询
python -m graphrag.query \
  --root ./my_project \
  --method global \
  "文档的核心主题是什么？"

# 📍 局部查询
python -m graphrag.query \
  --root ./my_project \
  --method local \
  "介绍一下 XX 实体的相关信息"
```
